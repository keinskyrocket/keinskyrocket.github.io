What are the differences between relative, absolute, and fixed positioning?

Sprint 1

What does position property do?

The position is a name of CSS property which put any HTML element at whatever location you like. You can specify whether you want the element positioned relative to its natural position in the page or absolute based on its parent element.

With applying the offset for the positioning properties: left, right, top, bottom, z-index, clip and position, you can specify the location of the HTML element.

Property values

There are seven position properties:

  • static - is the default value, which doesn't have any effect on the HTML element. The element will be displayed as the normal page flow.
  • relative - changes the position of the HTML element in the normal flow of the document. Unlike static elements
  • absolute - is removed from the normal document flow. The other elements will behave as if that element is not in the document.
  • fixed - is removed from the normal document flow like absolute, and allows you to fix the position of an element to a particular spot on the page, regardless of scrolling.
  • sticky - is positioned according to the normal flow of the document until the scroll location of the viewport reaches a certain point. And it acts like a fixed element.
  • initial - sets back to the brower default CSS position property when you have changed the property you wanted to be as your new default.
  • inherit - grabs the positioning value from its parent. Unlike initial, it doesn't grab the browser default property.

Each property serves a specific purpose. Let's take a closer took at three most frequestly used position's values in comparison with position: static;.

Elements with positioning as static are not affected by any of the CSS Positioning properties (left, right, top and bottom).

1. position: relative;

The positioning property for position: relative;(the yellow box) is set to top: 20px; and left: 50px;. It is moved 20px to the bottom direction and 50px to the right direction from the previous element which is the teal box. The yellow box follows the normal flow of the document as intended. The yellow box moves together with the rest of the elements when you are scrolling.

2. position: absolute;

The positioning property for position: absolute;(the yellow box) is set to top: 100px; and left: 200px;. It is moved 100px to the bottom direction and 200px to the right direction from the origin point of the page. The other elements ignore the yellow box as if it is not there. The yellow box moves together with the rest of the elements when you are scrolling.

See the Pen Example: position: absolute; by Keisuke (@keinskyrocket) on CodePen.

3. position: fixed;

The positioning property for position: fixed;(the yellow box) is set to top: 100px; and left: 200px;. The way it is positioned is the same as position: absolute;. However, the location of the yellow box remains as it is when you are scrolling.

Wrap up

The position: relative; could probably come handy when stylign a dropdown type of nagivation with multiple tiers. The lower tier must be tied to its parent all the time to keep its information architecture together.

When I want to isolate a HTML element from others and give something unique to it, I will use position: absolute; because its positioniing is pretty much independent. I'm thinking about cases like decorative objects to move around where I want them to be.

Lastly, the position: fixed; should widely be used for elements like sticky navigation that could ptentially brings better UX by making the component visible all the time.
