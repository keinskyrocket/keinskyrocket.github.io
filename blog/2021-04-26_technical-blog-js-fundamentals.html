if-else and switch

Let's take a look at the example of how if else and switch can be used within a context of everyday life with an example of psychic dog I've never had.

Example with if else
if (my dog barks once) {
  "Cuddle him"
} else if (my dog barks twice) {
  "Give him a snack"
} else if (my dog does NOT barks at all) {
  "Grab his toy and go for a walk"
} else {
  "Assume he is talking to a ghost"
}
Example with switch
switch (my dog) {
  case barks once:
    "Cuddle him" 
    break
  case barks twice:
    "Give him a snack"
    break
  case barks three times:
    "Grab his toy and go for a walk"
    break
  default:
    "Assume he is talking to a ghost"
}

So what's happening here is basically an if else is used for taking a true or false decision whereas a switch statement is used to test the condition against a list of case. As you read through the script in the if example, you must have answered yes or no to given conditions subconsciously. On the other hand, you must have chosen the most relevant answer to your situation in the switch example.

To give more solid understandings on them, here is a brief comparison between these two:

if else

  • The expression decides whether to execute the statements
  • Checks for equality and logical expression
  • Evaluates integer, character, pointer or floating-point type or boolean type
  • Difficult to edit and tedious to trace

switch

  • The expression decides which case to execute
  • Checks only for equality
  • Evaluates only character or an integer datatype
  • Much simpler to edit and easy to trace

while, do-while and for

These three are loops. Again, I was very confused with how to differentiate them. But the key to understand these three is to know when the statement is executed and how many times. Let's take a look at this example.

while (my crush is interested in me) {
  "Show my affection"
}

When you use while loop like above, your execution continues infinitely as long as you meet the condition.

do {
  "Show my affection"
} while (my crush is interested in me)

Like the example above, both while and do while can loop the execution forever while the condition is met. But there is a big difference between these two.

In while loop, I never try to show my affection while I know my crush is NOT interested in me, whereas I give it a shot only once in the do-while loop above while I know my crush is NOT interested in me. In terms of making an important life choice, I personally like to try at least once. However, I just need to be smart enough to know whether to use while or do-while when living in JavaScript world.

What about another loop, for?

for (12 years old now; my crush is interested in me till I turn 18; getting older) {
  "Show my affection"
}

This is like I somehow figured out my first love won't last long, and I see this teenage love is temporary. So I've decided to show my affection till I graduate high school. What a kid... Apart from how realistic I am (metaphorically speaking), I know how many times I want to loop the execution when I use for loop.

What is DOM?

Perhaps, enough with metaphors so far. I'll make this blog article sound a bit more academic! Let's have a look at this section a bit seriously.

DOM is a model that handles document as object. In other word, it is an interface to manipulate HTML/XML by programming. DOM allows us to address a specific part of files and send a command like "I want to do this action(event) at this part(object)." DOM also features:

  • creating a hierarchy called tree structure.
  • can be described in notes.
  • connects between Web page and programming language like JavaScript.

DOM allows us to not only change the contents of web page but also change styles. It can also specify notes that you want to modify while adding ID attribute as "mark".

Difference between accessing data from arrays and objects

I'll go over this section nice and easy. Simply put, arrays access data by referencing index whereas objects do so by referencing property.

const arr = ["Land", "Air", "Ocean"];
console.log(arr[0]);
const obj = {
  Strength: "Alcohol",
  Weakness: "Puppies",
  Charm: "Nail colour"
};
console.log(obj.name);

What are functions and why are they useful?

According to the definition of JavaScript found on W3Schools website, a JavaScript function is a block of code designed to perform a particular task, and is executed when "something" invokes it (calls it). To supplement my understanding it slightly better, I made a list of things that function can do below:

  • Group multiple commands into one
  • Pass multiple values
  • Return values
  • Create a scope
  • Reusable

This is at least what I know about JavaScript, and there should be more going on to the list! Here is an example of what difference function can make.

Before

This basically shows you an alert message saying "Greetings!" when you open the page. It is only one-time event.


alert("Greetings!");
After

When you want to add a bit of interactivity, here if what you can do. This code shows a button called Hit me!, and you will see an alert window saying "Greetings!" when you click the button. Instead of showing the alert window passively, you can proactively control a set of tasks such as:

  • Have a button on a page
  • Trigger a modal window by clicking the button
  • Display the modal window
  • Show a massage in the modal window

<button onclick="hitMe()">Hit me!!</button>
          
function hitMe() {
  var myText = "Greetings!";
  alert (myText);
}
